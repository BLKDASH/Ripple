# QML 串口助手需求文档

## 1. 项目概述

开发一款基于 Qt6 / QML 的桌面串口调试助手，用于在 PC 上与嵌入式设备、传感器、模组等进行串口通信调试。

- **目标平台**：Windows / Linux / macOS 桌面端
- **技术栈**：Qt 6.8+、QML、Qt Serial Port、C++ 后端模型
- **项目代号**：CWY

## 2. 功能需求

### 2.1 串口连接管理

| 配置项 | 可选项 | 默认值 |
|--------|--------|--------|
| 串口 | 枚举系统可用串口（名称 + 描述） | 第一个可用串口 |
| 波特率 | 1200 / 2400 / 4800 / 9600 / 19200 / 38400 / 57600 / 115200 / 230400 / 460800 / 921600 / 自定义 | 115200 |
| 数据位 | 5 / 6 / 7 / 8 | 8 |
| 停止位 | 1 / 1.5 / 2 | 1 |
| 校验位 | None / Even / Odd / Space / Mark | None |
| 流控 | None / Hardware(RTS/CTS) / Software(XON/XOFF) | None |

- 支持 **打开 / 关闭串口**。
- 打开串口时校验参数合法性，失败时给出明确错误提示。
- 串口状态实时显示：未连接 / 已连接 / 连接失败 / 发送中 / 接收中。

### 2.2 数据接收

- 串口数据实时显示在接收区。
- 显示模式切换：
  - **文本模式**：以 ASCII/UTF-8 解码显示。
  - **HEX 模式**：以十六进制字节流显示，字节之间用空格分隔。
- 可选自动换行（按行尾 `\n` 或固定时间/字节阈值换行）。
- 时间戳：可选在每一行前面加上 `[HH:mm:ss.zzz]` 时间戳。
- 接收区支持一键清空。
- 接收缓存上限：默认保留最近 10 000 行，超出自动丢弃最早数据（可配置）。

### 2.3 数据发送

- 发送内容输入框，支持：
  - 普通文本发送。
  - HEX 发送：输入 `01 02 03` 等十六进制字符串，校验合法性后发送。
- 发送选项：
  - 追加回车 `\r`。
  - 追加换行 `\n`。
  - 追加回车换行 `\r\n`。
- 支持定时循环发送：设置间隔 ms，勾选后周期性发送当前内容。
- 支持单次发送与快捷键发送（如 `Ctrl+Enter`）。
- 显示发送字节数 / 接收字节数统计，可清零。

### 2.4 快捷发送（多指令管理）

- 提供 8~16 组快捷发送按钮。
- 每组可自定义：
  - 按钮名称
  - 发送内容
  - 是否为 HEX
  - 是否追加换行
- 支持保存 / 加载快捷指令配置文件（JSON 格式）。

### 2.5 日志与文件

- 接收区内容可保存为文本文件（`.txt`）或二进制文件（`.bin`）。
- 支持打开文件并作为 HEX 或文本内容加载到发送框。
- 支持将接收数据自动写入日志文件（可选）。

### 2.6 界面与交互

- 主界面分为：
  - 顶部工具栏：打开/关闭串口、清空接收区、保存日志、设置。
  - 左侧：串口参数配置面板。
  - 中间：接收区（可滚动、可复制、可选择）。
  - 右侧/底部：发送区 + 快捷发送区。
- 深色 / 浅色主题切换。
- 窗口大小可调，最小尺寸 800×600。
- 状态栏显示：连接状态、端口名、波特率、RX 字节数、TX 字节数。

## 3. 非功能需求

- **性能**：波特率 921600 下接收稳定，不丢包、不卡顿。
- **稳定性**：串口异常拔出时自动关闭并提示用户。
- **跨平台**：同一套代码在 Windows、Linux、macOS 上均可编译运行。
- **可维护性**：QML 负责 UI，C++ 负责串口逻辑与数据模型，边界清晰。
- **国际化**：界面字符串使用 `qsTr`，预留中英文切换能力。

## 4. 技术方案

- **CMake 构建**：使用 `qt_add_qml_module` 组织项目。
- **C++ 后端**：
  - 封装 `SerialPortWorker` 或 `SerialPortManager` 类，继承 `QObject`。
  - 使用 `QSerialPort` 完成打开、关闭、读、写。
  - 通过信号 `dataReceived(QByteArray)` 将数据抛给 QML。
  - 通过 `Q_INVOKABLE` 暴露接口给 QML 调用。
- **QML 前端**：
  - 使用 `QtQuick.Controls` 构建控件。
  - 使用 `TextArea` 显示接收数据。
  - 使用 `ListModel` / `Repeater` 管理快捷发送按钮。
- **CMake 依赖**：
  ```cmake
  find_package(Qt6 REQUIRED COMPONENTS Quick SerialPort)
  target_link_libraries(appCWY PRIVATE Qt6::Quick Qt6::SerialPort)
  ```

## 6. 开发里程碑

| 阶段 | 目标 | 关键交付 |
|------|------|----------|
| M1 | 基础框架 | CMake 接入 SerialPort，C++ 串口类封装，QML 基础布局 |
| M2 | 核心收发 | 串口打开/关闭、参数配置、文本/HEX 收发、状态显示 |
| M3 | 增强功能 | 时间戳、自动换行、统计计数、定时发送、快捷发送 |

## 7. 验收标准

- [ ] 能正确枚举本机可用串口。
- [ ] 在 115200 / 8 / N / 1 下与常见串口设备正常双向通信。
- [ ] 文本/HEX 收发结果与预期一致。
- [ ] 拔出串口设备后程序不崩溃，并正确提示。
- [ ] 接收区大数据量（>1 MB）下不卡顿。
- [ ] 快捷发送、文件保存、主题切换功能可用。
- [ ] 在 Windows 11 与至少一个 Linux 发行版上编译运行通过。

## 8. UI/UX 设计

### 8.1 整体布局

采用经典的三栏 + 状态栏桌面工具布局，结构清晰、信息密度适中：

```
┌─────────────────────────────────────────────────────────────┐
│  工具栏（打开/关闭串口 | 清空 | 保存 | 设置 | 主题切换）       │
├──────────┬──────────────────────────────┬───────────────────┤
│          │                              │                   │
│  串口    │                              │    发送区         │
│  参数    │      接收显示区              │  （输入框+选项）   │
│  配置    │      （文本 / HEX）          │                   │
│  面板    │                              ├───────────────────┤
│          │                              │                   │
│          │                              │   快捷发送按钮区   │
│          │                              │   （8~16 按钮）   │
│          │                              │                   │
├──────────┴──────────────────────────────┴───────────────────┤
│  状态栏：状态 | 端口 | 波特率 | RX: 0 | TX: 0 | 光标位置       │
└─────────────────────────────────────────────────────────────┘
```

- **默认窗口尺寸**：1000 × 700。
- **最小窗口尺寸**：800 × 600。
- **响应式规则**：
  - 窗口宽度 < 1000 时，右侧发送区宽度收缩到最小 280 px；
  - 窗口高度 < 600 时，快捷发送区可折叠或改为两行显示。

### 8.2 顶部工具栏

位于窗口最上方，使用 `ToolBar` 实现，包含以下控件：

| 控件 | 功能 | 图标建议 |
|------|------|----------|
| 打开/关闭串口按钮 | 切换串口连接状态，连接中为红色断开图标，未连接为绿色连接图标 | 🔌 / ⏻ |
| 清空接收区按钮 | 清空中间接收显示区 | 🗑 |
| 保存日志按钮 | 将接收区内容保存为 `.txt` 或 `.bin` | 💾 |
| 加载发送文件按钮 | 将文件内容加载到发送框 | 📂 |
| 设置按钮 | 打开应用设置弹窗 | ⚙ |
| 主题切换按钮 | 在浅色/深色主题间切换 | 🌙 / ☀ |

- 按钮带 Tooltip，鼠标悬停显示功能说明。
- 串口打开后，除"关闭串口"外，部分串口参数控件应置灰不可修改（避免运行中误改）。

### 8.3 左侧串口参数配置面板

宽度固定 260 px，使用 `GroupBox` 分组，包含：

- **串口选择**：`ComboBox` 下拉框，显示 `COM3 (USB-SERIAL CH340)` 等可读名称；旁边放置刷新按钮 🔃。
- **波特率**：`ComboBox`，支持 editable，允许用户输入自定义波特率。
- **数据位 / 停止位 / 校验位 / 流控**：四个 `ComboBox` 下拉选择。
- **高级选项**（可折叠）：
  - 接收缓存行数上限
  - 是否显示时间戳
  - 是否自动换行
  - 是否在接收到 `\n` 时高亮整行

面板底部显示当前串口状态：
- 未连接：灰色圆点 + "未连接"
- 已连接：绿色圆点 + "已连接：COM3 @ 115200"
- 连接失败：红色圆点 + 错误简述

### 8.4 中间接收显示区

占据窗口主要区域，使用 `TextArea` 或自定义日志视图：

- **模式切换 Tab**：
  - 文本模式
  - HEX 模式
- **显示设置工具条**：
  - 是否显示时间戳
  - 是否自动换行
  - 是否暂停显示（暂停后数据仍在后台接收，恢复时一次性刷新）
  - 清空按钮
- **内容特性**：
  - 文本模式支持等宽字体（`Consolas` / `JetBrains Mono` / `Source Code Pro`）。
  - HEX 模式每 16 字节自动换行，便于阅读。
  - 滚动条自动跟随最新数据；用户手动向上滚动时自动锁定，到底部后重新自动跟随。
  - 右键菜单支持：复制、全选、清空、保存选中内容。

### 8.5 右侧发送区

分为上下两部分：

**上半部分：发送输入与选项**

- `TextArea` 多行发送输入框，支持文本/HEX 两种模式切换。
- 输入框下方放置选项：
  - `CheckBox`：HEX 发送
  - `CheckBox`：追加 `\r`
  - `CheckBox`：追加 `\n`
  - `CheckBox`：定时循环发送
  - `SpinBox`：定时间隔 ms（默认 1000）
- 发送按钮：大号主按钮，支持 `Ctrl+Enter` 快捷键。
- HEX 模式下输入非法字符时输入框边框变红，并在下方提示错误。

**下半部分：快捷发送区**

- 使用 `GridLayout` 或 `Flow` 排列 8~12 个快捷按钮。
- 每个按钮显示自定义名称，无名称时显示 `Cmd 1`、`Cmd 2` 等默认名。
- 右键按钮可编辑：名称、内容、是否为 HEX、是否追加换行。
- 按钮支持拖拽排序。
- 提供"保存配置"/"加载配置"入口。

### 8.6 底部状态栏

使用 `ToolBar` 或 `Rectangle` 实现，高度 28 px，显示：

- 左侧：连接状态指示灯 + 状态文字
- 中间：当前端口与参数，例如 `COM3 | 115200 8N1 None`
- 右侧：`RX: 12345 B | TX: 678 B`

所有文字使用小字号等宽字体，状态栏背景色与主题一致但略深于主背景。

### 8.7 对话框

- **设置对话框**：
  - 主题选择（浅色/深色/跟随系统）
  - 默认串口参数
  - 默认发送目录
  - 日志自动保存开关与目录
  - 接收缓存行数上限
- **快捷指令编辑对话框**：
  - 名称输入框
  - 内容输入框（文本/HEX 切换）
  - 追加换行选项
- **关于对话框**：
  - 应用名称、版本号、Qt 版本、开源许可链接
- **错误提示弹窗**：
  - 串口打开失败、权限不足、设备拔出、HEX 格式错误等。

### 8.8 视觉风格

- **设计语言**：扁平化、现代化桌面工具风格，参考 VS Code / Serial Port Utility。
- **配色方案**：
  - 浅色主题：背景 `#FFFFFF`，面板 `#F3F3F3`，边框 `#E5E5E5`，主蓝色 `#0078D4`。
  - 深色主题：背景 `#1E1E1E`，面板 `#252526`，边框 `#3C3C3C`，主蓝色 `#3794FF`。
- **字体**：
  - 界面字体：系统默认无衬线字体。
  - 数据区域字体：等宽字体，字号 12~14 px 可调。
- **圆角与间距**：控件圆角 4 px，面板间距 8 px，组间距 12 px。
- **图标**：使用 Qt 自带 `Material`/`Fluent` 图标或内嵌 SVG，确保浅色/深色下均清晰可辨。

### 8.9 交互动效

- 按钮点击有轻微按压反馈。
- 状态切换时指示灯颜色过渡动画 200 ms。
- 接收新数据时状态栏 RX 计数有短暂高亮脉冲。
- 连接失败时弹窗从顶部滑入，3 秒后自动消失或用户手动关闭。

### 8.10 可访问性

- 所有输入控件设置 `accessibleName` 与 `accessibleDescription`。
- 支持 Tab 键顺序切换焦点。
- 高对比度模式下文字与背景对比度符合 WCAG AA 标准。
